## 第2篇：Vibe Coding 复盘帖（长线收藏）—— 从0到1

### 标题
> 不会 Swift 的我把设计稿扔给了 AI，3 小时手搓了一个 macOS 软件

### 封面建议
**对比图**：左边是手画草图/初版原型，右边是最终成品截图。
- 或者：一行 Swift 代码截图 vs 菜单栏成品效果
- 左上角标注「3小时」 左下角标注「0行代码手写」

### 正文

作为一个不会 Swift 的人，我做了一个 macOS 菜单栏 app。

事情的起因很简单——Codex 经常在后台等我确认，但我切窗口过去才知道。于是就想要个红绿灯。

**第一步：想清楚要什么**
三个颜色，横排，老式信号灯风格，带呼吸动画。没有 PRD，就拿嘴说。

**第二步：AI 搭骨架**
「给我建一个 macOS 状态栏 app，用 .app bundle」

**第三步：踩坑和迭代**
- 红绿灯在绿色和红色之间闪 → 因为用了时间窗口猜状态，不靠谱
- 换成直接读 Codex 的 rollout 文件 → 还是有边界 case
- 最后发现 Codex 有现成的 hooks 机制 → 事件驱动，永不误判 ⚡️

**第四步：打磨细节**
菜单显示对话名、黄灯 8 秒弹通知并切到 Codex、红灯显示上次思考时长……这些全是 AI 写的，我负责说「不够复古」「灯再大点」「菜单太宽」。

**核心收获**
Vibe Coding 不是「AI 替你做」，而是「你当产品经理，AI 当工程师」。关键能力是：
- 知道什么是好产品（审美 + 体验判断）
- 能描述清楚你要什么
- 会排查问题、给正确方向

全部开源在 GitHub 👉 ooaaeiei2-beep/CodexTrafficLight

## 第3篇：开源叙事帖（品牌沉淀）—— 致敬 + 技术

### 标题
> 致敬两个开源项目，我搓了个 Codex 专属的 macOS 红绿灯 🚦

### 封面建议
**三栏并排**：左 agent-light 截图 / 中 codex-lamp 截图 / 右 CodexTrafficLight 成品
- 标题文字：「站在巨人的肩膀上」
- 或者在三个项目 logo/app 截图上叠加箭头表示演进关系

### 正文

最近做了一个小工具，核心思路来自两个很棒的开源项目 👇

**eternityspring/agent-light**
「用 hooks 事件驱动，别轮询，别猜」—— 就这一句话，我少走了三天弯路。

之前我写的方案是每秒读 Codex 的 rollout 日志文件、解析 JSON 字符串来判断状态。麻烦、延迟、还有边界 case。

**loopbrew/codex-lamp**
Codex 的 hooks.json 配置格式、状态文件通信模式、特别是 PermissionRequest 这个冷门但关键的事件——全是从这里学的。

他们做的是物理灯的蓝牙控制，我做的是菜单栏红绿灯。底层逻辑一模一样，只是 UI 层不同。

**我的版本加了什么？**
- 复古横排三灯 + 绿灯呼吸 / 黄灯急闪动画
- 菜单里直接看当前对话名（从 session_index 读）
- 黄灯 8 秒自动弹通知 + 点击通知切到 Codex
- 红灯显示上次思考时长
- 纯 Swift 单文件，编译即用

Swift 源码+编译脚本全在 GitHub 👉 ooaaeiei2-beep/CodexTrafficLight

开源不是闭门造车。感谢 agent-light 和 codex-lamp 的作者 ❤️
